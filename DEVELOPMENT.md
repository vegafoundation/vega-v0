# VEGA-v0 Development Workflow

## Branches:
- main: Live/Production (geschützt)
- development: Neue Features

## Workflow:
1. In development arbeiten
2. Änderungen committen
3. Pushen
4. Pull Request erstellen
5. Nach Review in main mergen

## Befehle:
git checkout development    # Zu development wechseln
git checkout main          # Zu main wechseln
git checkout -b feature/xyz # Neue Feature-Branch
